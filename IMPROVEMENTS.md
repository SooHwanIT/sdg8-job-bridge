## 2. 기술 하이라이트 — 근로기준법 규칙엔진

계약 검토를 "그럴듯한 텍스트"가 아니라 **실제로 법을 코드로 옮겨 실행하는 컴플라이언스 체커**로 만들었습니다. 클라이언트 사이드 전용, 외부 API·키 없음.

### 2.1 구조 (Single Source of Truth)

```
src/lib/laborLaw.js       상수 · 조문 메타 · 룰셋(12개 규칙)
src/lib/contractEngine.js 정형 계약(contract) → 룰 실행 → {findings, score, grade}
src/data/db.js            각 공고의 contractData(정형 필드)  ← 유일한 진실
```

공고 카드·상세 계약 탭·"직접 검토" 폼이 **모두 같은 엔진**을 호출합니다. 데이터와 판정 로직이 한 곳에 모여 있어 일관되고, 규칙을 하나 고치면 앱 전체가 동일하게 반영됩니다.

### 2.2 룰(rule) 모델

각 규칙은 순수 함수 두 개로 구성됩니다.

```js
{
  id, article,             // 근거 조문(법·조·이름)
  applies: (c) => boolean,  // 이 계약에 규칙이 적용되는가
  run: (c) => ({ status, title, value, message }) // pass | warn | violation
}
```

엔진은 적용 대상 규칙만 실행 → 판정 목록(findings)을 만들고, 상태별 가중치(위반 24 / 주의 8 / 적법 0)를 100점에서 차감해 **준수 점수**와 **등급(안전·검토·주의·위험)**을 산출합니다. 문제가 먼저 보이도록 위반 → 주의 → 적법 순으로 정렬합니다.

### 2.3 인코딩한 실제 근로기준법 체크 (12종)

| 규칙 | 근거 | 판정 로직 요지 |
|------|------|----------------|
| 최저임금 | 최저임금법 §6 | 시급 < **10,030원(2025)** → 위반, 차액 표시 |
| 주휴수당 | 근기법 §55 | **주 15시간 이상**인데 미지급 → 위반 / 미만이면 비대상 |
| 서면 근로계약서 | 근기법 §17 | 구두계약 → 위반(500만원 이하 과태료 안내) |
| 연장 가산 | 근기법 §56 | 통상임금 **150%** 미만 → 위반 |
| 야간 가산 | 근기법 §56 | 22:00~06:00 **50% 가산** 미만 → 위반 |
| 휴일 가산 | 근기법 §56 | 8h 이내 50% / 초과 100% 미만 → 위반 |
| 수습 감액 | 최임법 시행령 §3 | 하한 **90%**, **1년 미만 계약은 감액 불가** → 위반 |
| 위약 예정 금지 | 근기법 §20 | 위약금·손배 예정 조항 → 위반(강행규정) |
| 재량근로 대상 | 근기법 §58③ | 법정 6개 업무 아님 → 위반 / 대상이어도 서면합의 확인(주의) |
| 연차 명시 | 근기법 §60 | "무제한 휴가" 등 법정 연차 미명시 → 주의 |
| 경업금지 | 민법 §103·판례 | 무보상 장기 제한 → 주의 |
| 4대보험 | 고용보험법 등 | 15h 이상인데 산재만/누락 → 주의, 프리랜서 근로자성 → 주의 |

### 2.4 "내 계약서 직접 검토" (와우 포인트)

`components/ContractChecker.jsx` — 사용자가 시급/근무시간/수당/수습/조항을 입력하면 `useMemo`로 **입력 즉시** 엔진이 재실행되어 점수·판정이 실시간 갱신됩니다.

- 프리셋 3종(최저임금 미달 / 위약금 계약 / 정상 야간 알바)으로 위험 계약을 원클릭 시연
- 토글·숫자 입력은 `role="switch"`, `aria-label`, 포커스 링 적용
- 조건에 따라 하위 입력이 나타나는 **동적 폼**(연장 있음 → 가산율, 재량근로 → 대상 업무 여부)

검증 결과(실데이터 8개 공고): 온기복지재단·카페온도·편의점 100점(안전), 누리테크 92(무보상 경업금지), 한빛제조 76(위약예정 위반), 초록바이오 44(재량근로 오남용·연장수당 미지급·연차 미명시), 한빛물류 36(구두계약·주휴 미지급) — **의도한 스펙트럼대로 판정**됩니다.

## 4. UX 폴리시

- **반응형**: `md` 이상 좌측 사이드바(회사=블루/알바=앰버 테마 유지), 이하 하단 탭. 강한 모드 게이트 유지.
- **로딩/전환**: 하이드레이션 스플래시, NEIS 불러오기 스켈레톤, 채팅 타이핑 인디케이터, 진입 애니메이션(`anim-in`), 점수 게이지 채우기 전환.
- **빈 상태**: 검색·지원·영상 없음 전용 컴포넌트.
- **접근성**: 다이얼로그 `role`/`aria-modal`, ESC 닫기, 탭 `role="tablist"`, 스위치/라디오 시맨틱, 포커스 링, `prefers-reduced-motion` 전면 존중.

## 5. AI 자소서 개선

`lib/coverLetter.js` — 불러온 프로필(자격증·출결·성적·수상)을 재료로 **도입-역량-태도-경험-포부** 5문단을 구성. 어조 3종(성실/성장/협업)과 지원 대상 입력을 반영하고, 생성 후 자유 편집 가능. 규칙/템플릿 기반이라 외부 추론 API가 필요 없습니다.

## 6. 저장/하이드레이션

`lib/storage.js` — `{ v: 2, data }` 스키마, v1(`{mode, applied}`) 자동 마이그레이션, `localStorage` 접근 실패(시크릿/차단) 시 메모리 폴백. 하이드레이션 완료 전에는 저장하지 않아 초기값 덮어쓰기를 방지. **지원 목록·프로필·자소서**를 영속화.

## 7. 심사 어필 포인트

1. **기술 깊이** — 근로기준법을 12개 실행 가능 규칙으로 인코딩한 규칙엔진. 조문 인용·점수화·실시간 판정까지, 데모가 아니라 동작하는 컴플라이언스 로직.
2. **UX 완성도** — 회사/알바 분리 컨셉을 지키면서 데스크톱까지 확장, 로딩·빈상태·접근성·모션 배려.
3. **사회적 임팩트** — 계약 정보 비대칭 해소로 첫 일자리·청소년 노동자를 착취적 계약에서 보호(SDG 8.5/8.6/8.8). 정성적 근거로 정직하게 서술.

## 8. 한계 · 향후

- 규칙엔진은 대표적 조항 중심의 **교육용**입니다. 실제 사건은 사실관계·판례에 따라 달라지므로 법률 자문이 아닙니다(고용노동부 ☎ 1350 안내).
- NEIS·전화·메시지·쇼츠는 목업/시뮬레이션입니다.
- 향후: 계약서 이미지 OCR → 필드 자동 추출, 지역 고용센터 연계, 규칙 커버리지 확대(퇴직금·5인 미만 사업장 특례 등), 다국어(이주노동자) 지원.
